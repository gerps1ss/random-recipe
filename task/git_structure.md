# Структура git-репозитория

## Файловая структура git-репозитория

```text
/
├── README.md
├── docs/
│   └── README.md
├── reports/
│   └── README.md
├── site/
│   ├── index.html
│   ├── recipes.html
│   ├── ingredients.html
│   ├── about.html
│   ├── login.html
│   ├── css/
│   │   └── style.css
│   └── images/
├── task/
│   ├── README.md
│   ├── git_structure.md
│   └── terms.md
└── .gitignore
```

## Описание структуры репозитория

### README.md

Содержит основную информацию о проекте:

- название проекта;
- участники;
- описание проекта;
- период выполнения;
- структура проекта.

### docs/

Документация проекта в формате Markdown.

### reports/

Отчёты по проектной (учебной) практике.

### site/

Исходный код веб-сайта.

Сайт предназначен для поиска случайных рецептов на основе продуктов, имеющихся у пользователя.

Основные страницы сайта:

- Главная страница;
- Страница рецептов;
- Страница ингредиентов;
- Страница «О проекте»;
- Страница авторизации.

### task/

Материалы задания и описание структуры репозитория.

### .gitignore

Файл для исключения временных файлов из Git.

## Преимущества структуры

- Чёткое разделение материалов проекта.
- Удобная навигация по репозиторию.
- Простое расширение проекта новыми файлами.
- Отдельное хранение документации, сайта и отчётов.
