# Структура git-репозитория

## Файловая структура git-репозитория

```text
/
├── README.md
├── docs/
│   └── README.md
├── reports/
│   └── README.md
├── site/
│   ├── index.html
│   ├── recipes.html
│   ├── about.html
│   ├── team.html
│   ├── journal.html
│   ├── resources.html
│   ├── style.css
│   ├── script.js
│   └── images/
│       ├── logo.png
│       ├── salad.png
│       ├── resources-bg.png
│       ├── member1.png
│       ├── member2.png
│       ├── recipe_photo/
│       ├── about/
│       └── journal/
├── task/
│   ├── README.md
│   └── git_structure.md
└── .gitignore
```

## Описание структуры репозитория

### README.md

Содержит основную информацию о проекте:

* название проекта;
* участников проекта;
* описание проекта;
* цели и задачи;
* используемые технологии;
* структуру репозитория.

### docs/

Папка с документацией проекта в формате Markdown.

Содержит:

* описание проекта;
* описание страниц сайта;
* инструкции по использованию;
* дополнительные материалы.

### reports/

Папка для отчётов по учебной практике.

Содержит:

* промежуточные отчёты;
* итоговый отчёт;
* результаты выполнения этапов проекта.

### site/

Исходный код веб-сайта.

Сайт предназначен для поиска рецептов на основе имеющихся у пользователя ингредиентов.

Основные страницы сайта:

* `index.html` — главная страница;
* `recipes.html` — каталог рецептов;
* `about.html` — информация о проекте;
* `team.html` — участники проекта;
* `journal.html` — журнал разработки;
* `resources.html` — полезные ресурсы.

Дополнительные файлы:

* `style.css` — стили сайта;
* `script.js` — интерактивность и логика работы;
* `images/` — изображения, фотографии и иконки.

### site/images/

Папка с графическими материалами проекта:

* логотип проекта;
* фотографии рецептов;
* изображения для оформления страниц;
* фотографии участников;
* иконки и графические элементы интерфейса.

### task/

Материалы задания и описание структуры проекта.

Содержит:

* `README.md` — описание задания;
* `git_structure.md` — описание структуры репозитория.

### .gitignore

Файл для исключения временных, служебных и автоматически создаваемых файлов из системы контроля версий Git.

## Преимущества структуры

* Чёткое разделение исходного кода, документации и отчётов.
* Удобная навигация по репозиторию.
* Простое сопровождение и расширение проекта.
* Отдельное хранение изображений и графических материалов.
* Подготовленная структура для публикации проекта через GitHub Pages.
